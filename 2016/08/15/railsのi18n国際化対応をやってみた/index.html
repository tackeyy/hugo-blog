RailsのI18n（国際化対応）をやってみた

2016-08-15 · 70 words · 1 minute read
rails
rails · local · i18n

今回は国際化ではなくてviewに文字をベタ書きしたくないので、
i18nを使用して config/locales/ja.yml に定数をまとめました。

i18nの方法はRailsガイドのここに記載されている通りです。
要点だけ以下にまとめます。

もくじ

  1. default localを:jaにする
  2. ja.ymlのテンプレをDLする
  3. ja.ymlに定義した定数を呼び出してみる
  4. 雑談

default localを:jaにする

特に設定を行っていない場合は、config/application.rb が以下のようになっています。
# config.i18n.default_locale = :de
日本語にする場合は以下のようにコメントを外してシンボルを変更します。
config.i18n.default_locale = :ja

ja.ymlのテンプレをDLする

以下のテンプレをDLします。
内容はここから確認できます。

wget https://raw.github.com/svenfuchs/rails-i18n/master/rails/locale/ja.yml -P config/locales/

wget をインストールしておらず Mac OSを使っている場合は、
以下のコマンドで wget をinstallしてください。

brew install wget

default localを:jaにする

bundle exec rails c で動作確認してみます。

I18n.t('date.abbr_day_names') pry(main)> I18n.t('date.abbr_day_names') => ["日", "月", "火", "水", "木", "金", "土"]

雑談

フォームの追加、詳細などは別のymlファイルにして管理しようとしたのですが、
（ I18n.t('form.buttons.show') のようなイメージ ）
config/locales/form/ja.yml を作成して、
config.i18n.load_path を変更してみてもダメでした..
modelごとにymlファイルを分割して管理できるのでできそうなのに。
